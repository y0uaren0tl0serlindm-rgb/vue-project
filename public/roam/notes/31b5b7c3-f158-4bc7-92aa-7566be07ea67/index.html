#include<stdio.h>

int main(void){
  printf("Hello ARM!\n");
  return 0;
}

在普通的情况下我们使用普通的gcc编译 会得到当前的cpu架构的程序 比如 x8664 服务器 得到 x8664 Linux 程序 

gcc hello.c -o hello

然而我们需要生成能在ARM架构上能运行的程序 于是我们使用arm-linux-gcc 得到 ARM Linux 程序 

arm-linux-gcc hello.c -o hello

1. ARM-Linux-gcc

arm-linux-gcc hello.c -o hello 一系列流程如下

hello.c ⬇预处理 展开后的c代码 ⬇编译 ARM 汇编 ⬇汇编 ARM目标文件 hello.o ⬇链接 ARM Linux 可执行文件 hello

1.1. 预处理

arm-linux-gcc -E hello.c -o hello.i

得到hello.i

这里会将代码种的#include <xxx.h>展开进去

1.2. 编译

arm-linux-gcc -S hello.c -o hello.s

得到hello.s

这里面是ARM汇编，不是x86汇编 里面会有ARM指令

1.3. 汇编

arm-linux-gcc -c hello.c -o hello.o

得到hello.o

这是ARM目标文件，还不能直接运行

1.4. 链接

arm-linux-gcc hello.o -o hello

通过动态链接 得到hello

完整的ARM Linux可执行程序

1.4.1. 为什么链接这一步很重要？

例如，代码中有这么一段 printf("fuck you Apex\n");

这不是我写的，而是来自C library也就是这次工具链里的musl

hello.o + musl中的printf + 启动文件 + 动态链接器信息 = 最终 hello 程序
