Makefile内容 

obj-m += hello.o

all:
make -C /lib/modules/$(shell uname -r)/build M=$(PWD) modules

clean:
make -C /lib/modules/$(shell uname -r)/build M=$(PWD) clean

1. 解读

1.1. obj-m += hello.o

这个代码意思是 我有一个目标叫hello.o，请按 模块 (m for module)的规矩处理它 kbuild看到这行就会自动找对应的hello.c去编 如果写 obj-y 就变成了按 内置代码 编，这会把代码便进内核本体

1.2. make -C /lib/modules/\((shell uname -r)/build M=\)(PWD) modules

1.2.1. -C

先 cd 进入这个目录再执行make。走进Linux内核源码树 为啥要进去？因为编内核模块的所有规矩、模板、头文件路径，都在内核源码树里现成准备好了。你不用自己重造这套东西，借用就行。

1.2.2. M=$(PWD)

进入源码树后，再告诉kbuild:"真正要编的源文件在 M 这个路径下"即当前目录

1.2.3. module

是 kbuild 内置的目标名，意思是"按外部模块的方式构建" 合起来这句话是："借用内核源码树的工具和规矩，去编我这边的代码。"

1.3. make -C /lib/modules/\((shell uname -r)/build M=\)(PWD) clean

1.3.1. clean

同理，借用内核的清理规则。
