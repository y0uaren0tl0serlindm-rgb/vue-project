1.3. CPU复位

什么是CPU复位？ 为什么CPU会执行固件代码？ 为什么固件代码是0xFFFFFFF0？

  • 具体流程
    • 释放 RESET 信号后 CPU 被释放 在收到这个信号之前 CPU 会一直保持在一个叫做 RESET 的状态等待电源和时钟稳定
    • CPU 被释放后会进入复位后的初始状态 也就是一种固定状态 寄存器被设为初始值 缓存、流水线等处于初始状态 CPU 开始从固定地址取指令
    • CPU 从固定地址 0xFFFFFFF0(Reset Vector 复位向量) 取第一条指令
    • 芯片组 把这个地址映射到 BIOS/UEFI Flash ROM
    • CPU 执行固件入口跳转代码
